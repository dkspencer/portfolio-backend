# Introduction

This Portfolio API is built on HTTPS and is RESTful. The Portfolio API accepts and returns JSON in the request body.

You can use any HTTPS/REST libraries with a variety of programming languages.

# Errors

The API uses standard HTTP status codes to indicate the success or failure of an API call. The body of the response will be JSON and follow this format (with a different "status_code" and "non_field_errors" ): 


  {
    "status_code":400,
    "errors": {
      "non_field_errors": [
        "Invalid input."
      ]
    }
  }
